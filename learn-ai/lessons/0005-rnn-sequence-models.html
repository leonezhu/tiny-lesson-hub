循环神经网络 RNN 与序列模型

第五课 · ~12 分钟 · 序列数据处理

本课结束时，你将能够：理解为什么普通神经网络无法处理序列数据，掌握 RNN 的"记忆"机制，知道 LSTM 如何解决长期依赖问题，并能说出 RNN/LSTM 在现实中的典型应用场景。

为什么需要 RNN？

前面我们学过的神经网络（感知机、CNN）都有一个共同特点：输入是独立的、固定的。一张图片是一张图片，另一张图片是另一张，它们之间没有关系。

但现实世界中很多数据是序列——前后顺序很重要：

序列数据的特征

  • 文本："我吃饭" ≠ "饭吃我"。词序决定含义。
  • 语音：音素按顺序排列才构成有意义的音节。
  • 时间序列：股票价格今天受昨天的影响，昨天受前天的影响。
  • 视频：每一帧与前一帧、后一帧紧密相关。
  • DNA：碱基对的排列顺序决定了遗传信息。
关键洞察：普通神经网络（如 CNN）处理每张输入时都是"从头开始"，它没有"记住"之前看到的东西的能力。但处理序列时，过去的信息会影响对现在的理解。比如你读到"虽然……"，你必须记住这个词才能理解后面跟着的"但是……"。

RNN 的核心思想：带记忆的神经网络

RNN（Recurrent Neural Network，循环神经网络）解决这个问题的方法很简单而优雅：让网络有一个"隐藏状态"（hidden state），这个状态会随时间传递。

时刻 t=1: 输入"我" → 输出"喜欢" → 隐藏状态 h₁（记住了"我"）
↓
时刻 t=2: 输入"吃" + h₁ → 输出"饭" → 隐藏状态 h₂（记住了"我吃"）
↓
时刻 t=3: 输入"饭" + h₂ → 输出"了" → 隐藏状态 h₃（记住了"我爱吃饭"）

换句话说，RNN 在每个时刻做两件事：

RNN 的两个操作

  • 更新记忆：把当前输入和之前的隐藏状态结合起来，产生新的隐藏状态
  • 产生输出：基于当前输入和当前记忆，做出预测或分类

数学上可以写成两个公式：

RNN 的数学表达

ht = tanh(Whh · ht-1 + Wxh · xt)
yt = Why · ht

  • xt = 时刻 t 的输入（比如第 t 个词的词向量）
  • ht = 时刻 t 的隐藏状态（"记忆"）
  • yt = 时刻 t 的输出
  • Whh = 隐藏到隐藏的权重（记忆如何传递）
  • Wxh = 输入到隐藏的权重（输入如何影响记忆）
  • Why = 隐藏到输出的权重（记忆如何产生结果）
直觉理解：想象你在读一句话。每读一个字，你的大脑就更新一次对整句话的理解。这个"不断更新的阅读理解"就是隐藏状态 ht。RNN 的公式只是在模仿这个过程。

RNN 的三种典型架构

1. 双向 RNN（Bi-directional RNN）

有时候你需要"上下文"——不仅要看前面，还要看后面。比如情感分析：

"这部电影真的不好看"
只看前面的词 → 可能判断为正面（"真的"）
同时看后面的词 → 正确判断为负面（"不好看"）

双向 RNN 同时从前向后和从后向前处理序列，最后把两边的信息合并。

2. Encoder-Decoder（编码器-解码器）

这是机器翻译的基础架构：

编码器（Encoder）：输入"我喜欢猫" → 压缩成一个向量
解码器（Decoder）：从这个向量 → 输出"I love cats"

编码器把整个输入序列压缩成一个"意义向量"，解码器再从这个向量逐步生成输出序列。这就是早期机器翻译系统的核心思路。

3. 序列到序列（Seq2Seq）

Encoder-Decoder 的泛化形式，不仅用于翻译，还用于：

  • 文本摘要：长文章 → 简短总结
  • 对话系统：用户的问题 → 机器人的回答
  • 语音识别：音频序列 → 文字序列

RNN 的问题：长期依赖困境

RNN 理论上可以记住任意长的序列，但实际中遇到了一个严重问题：梯度消失（Gradient Vanishing）。

什么是梯度消失？

训练神经网络时，误差从输出层往回传（反向传播）。在 RNN 中，误差要穿过每一个时间步。如果每一步的权重矩阵相乘后数值越来越小，传到最前面的误差就会接近于零——网络"忘记"了早期的输入。

比喻：就像玩"传话游戏"。第一个人说"我今天早上吃了三个苹果然后去公园跑步最后看了场电影"，传到第十个人时可能只剩"吃苹果"了。中间的细节全部丢失。

句子："我从小在巴黎长大，___说一口流利的法语"
RNN 在填空白时：可能已经忘了"巴黎"这个词，因为中间隔了太多字
理想情况：应该记得"巴黎"，填入"他"

LSTM：长短期记忆网络

LSTM（Long Short-Term Memory）是 RNN 的一种改进版本，由 Hochreiter 和 Schmidhuber 在 1997 年提出。它引入了门控机制（gating mechanism），让网络可以主动选择"记住什么、遗忘什么"。

LSTM 的三个门

1. 遗忘门（Forget Gate）— "我该忘掉什么？"

决定隐藏状态中哪些信息应该被丢弃。比如你读到新句子了，旧句子的记忆就该清除。

ft = sigmoid(Wf · [ht-1, xt] + bf)
输出 0~1 之间的值：0 = 完全遗忘，1 = 完全保留

2. 输入门（Input Gate）— "我该记住什么新信息？"

决定哪些新信息应该加入细胞状态。包含两部分：

  • 门控部分：sigmoid 层决定哪些值需要更新
  • 候选部分：tanh 层创建新的候选值

it = sigmoid(...)   |   C̃t = tanh(...)
两者相乘 → 只加入被"选中"的新信息

3. 输出门（Output Gate）— "我该输出什么？"

基于当前的细胞状态，决定输出什么。

ot = sigmoid(...)   |   ht = ot · tanh(Ct)
细胞状态经过 tanh 压缩到 -1~1，再乘以输出门的值

LSTM 细胞状态（Cell State）：一条"高速公路"贯穿所有时间步
x1 ────[细胞状态]────> x2 ────[细胞状态]────> x3 ────> ...
每个时间步只有少量的线性交互（加减），信息可以几乎无损地传递很远的距离
这就是 LSTM 解决梯度消失的关键！
核心区别：普通 RNN 的记忆是"模糊的"——每个新输入都会修改记忆。LSTM 的记忆是"结构化的"——有专门的通道（细胞状态）存储长期信息，有专门的门控制何时写入和读取。这让它能记住几个月前的上下文。

GRU：门控循环单元

GRU（Gated Recurrent Unit）是 LSTM 的简化版，由 Cho 等人在 2014 年提出。它将遗忘门和输入门合并为一个更新门（Update Gate），计算更快，效果相近。

LSTM vs GRU 对比

  • LSTM：3 个门（遗忘、输入、输出），2 个状态（隐藏状态 + 细胞状态）
  • GRU：2 个门（更新门、重置门），1 个状态（隐藏状态）
  • 实际选择：数据量大时用 LSTM（表达能力更强），数据量小时用 GRU（不易过拟合，训练更快）

RNN/LSTM 的实际应用

自然语言处理

  • 机器翻译：Google Translate 早期版本使用 Encoder-Decoder LSTM
  • 文本生成：自动补全、写作助手（ChatGPT 的前身 GPT 系列也使用了 RNN 思想）
  • 命名实体识别：从文本中提取人名、地名、机构名
  • 情感分析：判断评论是正面还是负面

其他领域

  • 语音识别：将音频波形转换为文字（DeepSpeech 等系统）
  • 音乐生成：生成旋律、和弦进行
  • 股票预测：分析时间序列的价格模式
  • 视频分析：理解视频中动作的时序变化
  • 蛋白质结构预测：分析氨基酸序列
重要认知：虽然 Transformer 已经在 NLP 领域取代了 RNN/LSTM 成为主流（见第六课），但 RNN 的思想仍然深刻影响着 AI 的发展。而且在某些场景（如实时流数据处理、内存受限的边缘设备）中，RNN/GRU 仍然有其优势。

与前几课的关联

RNN 是整个 AI 课程的重要转折点：

  • 从 CNN 到 RNN：CNN 处理空间数据（图像），RNN 处理时间数据（序列）。它们是互补的。
  • 从反向传播到序列反向传播：RNN 使用 BPTT（Backpropagation Through Time），是反向传播在时间维度上的扩展。
  • 通向 Transformer：RNN 的长期依赖问题直接催生了注意力机制（Attention），而 Attention 是 Transformer 的核心——这就是第六课的内容。
