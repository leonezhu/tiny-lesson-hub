深度学习与反向传播

第三课 · ~12 分钟 · 神经网络的核心算法

学完这节课后，你将能够：
1. 解释为什么"深度"是神经网络能力飞跃的关键
2. 逐步推导反向传播的链式法则（Chain Rule）
3. 描述梯度下降如何更新权重
4. 理解学习率、批量大小、过拟合等核心超参数的直觉含义

从"浅"到"深"——为什么需要很多层？

上一课我们学到，一个隐藏层的前馈神经网络就能逼近任何连续函数（万能近似定理）。那为什么我们还要研究"深度"学习？为什么要把网络堆到几十层、几百层甚至上千层？

答案在于表示的效率。一个深层网络可以用指数级更少的神经元来表达一个浅层网络需要表达的功能。想象一下：如果浅层网络需要一个包含 1000 个神经元的隐藏层来识别图像中的"圆形"，那么一个深层网络可能只需要两层、每层 10 个神经元就能做同样的事情——因为第一层可以学到"边缘"，第二层可以把边缘组合成"曲线"，自然就得到了"圆形"。

深度网络的层次化学习

  • 第 1-2 层：学习低级特征——边缘、角点、颜色渐变（在图像中）；词根、短语（在文本中）
  • 第 3-5 层：组合低级特征——形状、纹理、词组结构
  • 第 6-10 层：高级语义——物体的部分（车轮、眼睛）、概念的组合
  • 最后几层：任务特定的决策——"这是一只猫" vs "这是一辆汽车"
浅层网络 输入 → 隐藏层（很多神经元） → 输出

深层网络 输入 → 层1 → 层2 → 层3 → ... → 层N → 输出
类比理解：想象你要判断一幅画是不是毕加索的作品。一个"浅层"方法可能要求你记住毕加索所有画作的像素级模板——这需要海量的记忆。而一个"深层"方法会让你先看"有没有不规则的形状"，再看"有没有立体主义的特征"，再看"色彩和构图是否独特"——层层深入，每一步都只问一个简单的判断，最终组合起来就能做出准确的判断。这就是深度学习的力量。

反向传播——神经网络如何"学习"

这是整节课的核心，也是深度学习之所以可能的基础算法。我们已经知道：前向传播是用当前权重做预测；损失函数告诉我们预测有多错。那下一步呢——我们怎么知道"该往哪个方向调整权重"才能让损失变小？

反向传播（Backpropagation）给出了精确的答案。它的核心思想是数学中的链式法则（Chain Rule）：如果一个变量 A 影响 B，B 影响 C，那么 A 对 C 的影响 = A 对 B 的影响 × B 对 C 的影响。

反向传播的三步走

  • 第一步：前向传播——输入数据通过网络，计算每一层的输出，最终得到预测值和损失。
  • 第二步：反向传播梯度——从输出层开始，沿着网络"往回走"，用链式法则计算损失对每一层每个参数的偏导数（梯度）。
  • 第三步：更新权重——沿着梯度的反方向更新权重，使损失减小。

链式法则——反向传播的数学引擎

让我们用一个极简的例子来理解反向传播是如何工作的。假设我们有一个三层的网络：

网络结构
输入 x → 加权 + 激活 → h → 加权 + 激活 → 输出 ŷ → 损失 L

具体公式
h = ReLU(w₁ · x + b₁)
ŷ = w₂ · h + b₂
L = (ŷ - y)² （均方误差）

现在我们的目标是：损失 L 对 w₁ 的偏导数是多少？也就是说，如果我把 w₁ 稍微增大一点，损失会怎么变化？

用链式法则分解
∂L/∂w₁ = ∂L/∂ŷ · ∂ŷ/∂h · ∂h/∂w₁

逐步计算
∂L/∂ŷ = 2(ŷ - y) （损失的导数）
∂ŷ/∂h = w₂ （ŷ 对 h 的导数）
∂h/∂w₁ = x · ReLU'(w₁·x + b₁) （ReLU 的导数：输入 > 0 时为 x，否则为 0）

最终结果
∂L/∂w₁ = 2(ŷ - y) · w₂ · x · ReLU'(w₁·x + b₁)
关键直觉：链式法则把"一个遥远参数的影响"分解成了几步"近处参数的影响"的乘积。每一步都只关心"这一步的微小变化对下一步的影响"，然后把这些"近处影响"乘起来，就得到了"远处的影响"。这就是反向传播的精髓——它不需要猜测，不需要暴力枚举，只需要一步一步地"往回推导"。

梯度下降——沿着梯度下山

有了梯度（损失对每个参数的偏导数），我们就可以更新权重了。最基础的方法是随机梯度下降（SGD）。

权重更新公式
w_new = w_old - learning_rate · ∂L/∂w

直觉理解
梯度 ∂L/∂w 告诉我们在哪个方向损失增长最快。为了最小化损失，我们要沿着梯度的反方向走一步。"学习率"（learning_rate）决定了这一步有多大。

学习率——最关键超参数的直觉

  • 学习率太大（如 0.1）——每一步跨得太远，可能跳过最低点，甚至发散。
  • 学习率太小（如 0.0001）——每一步都很谨慎，但需要数百万步才能到达最低点，训练极其缓慢。
  • 合适的学习率（通常 0.001 ~ 0.01）——稳步下降，在合理的步数内收敛。
学习率对训练的影响 L（损失）
↑                       *\n                        |\n                        |\n                  · · · ·       ← 太小：缓慢但稳定\n                ·               ← 太大：跳过最低点\n                          |\n                          ·\n                                        → 训练步数

批量训练与 Mini-Batch

在现实中，我们不会用单个样本来更新权重（那叫纯随机梯度下降），也不会一次用全部数据（那叫批量梯度下降，在大数据集上计算量巨大）。折中的方案是 Mini-Batch 梯度下降——每次取一小批数据（比如 32、64 或 128 个样本）来计算梯度和更新权重。

批量大小（Batch Size）的权衡

  • 小批量（16-32）——梯度噪音大，但每步更新快。噪音反而有帮助：它让梯度下降不会陷入尖锐的局部最优，更容易找到"平坦的"、泛化能力更好的解。
  • 中等批量（128-256）——大多数深度学习任务的标准选择。在稳定性和效率之间取得了良好的平衡。
  • 大批量（1024+）——GPU 利用率高，每轮训练快。但梯度估计更精确，噪音更少，可能泛化能力稍差（这个现象被称为"批量大小gap"）。

过拟合——深度学习最著名的挑战

想象一个学生为了考试而死记硬背所有习题的答案——考试时如果遇到一模一样的题目，他能满分通过。但如果题目稍微变了一下，他就完全不会了。这就是过拟合（Overfitting）：模型在训练数据上表现很好，但在未见过的数据上表现很差。

防止过拟合的常见方法

  • 数据增强（Data Augmentation）——对训练数据进行变换（图像翻转、裁剪、加噪声），人为扩充数据集，让模型见到更多"变体"。
  • Dropout——在训练时随机"关掉"一部分神经元（设为 0），迫使网络不要过度依赖任何单个神经元，增强鲁棒性。测试时所有神经元都打开。
  • 早停（Early Stopping）——监控验证集上的性能，当验证集性能不再提升时停止训练。防止模型在训练集上"学过头"。
  • 权重衰减（L2 Regularization）——在损失函数中加入权重的平方和惩罚项，迫使权重保持较小，降低模型复杂度。
一个有趣的观察：深度学习中最强大的模型往往参数量远超训练数据的"信息容量"。理论上它们应该严重过拟合，但实际上它们泛化得很好。这个现象被称为"深度学习的悖论"，至今仍在研究中。一个可能的解释是：梯度下降本身就是一个隐式的正则化器——它自然地倾向于找到"最简单的"解（参数小的解），即使模型有巨大的容量。

优化器的进化——从 SGD 到 Adam

基本的 SGD 是最简单的优化器，但它有一些明显的局限性：它使用统一的学习率更新所有参数，对每个参数的"学习速度"是一视同仁的。而在现实中，不同的参数可能需要不同的学习速度。

主流优化器对比

  • SGD（随机梯度下降）——基础版。简单、可解释，但不考虑历史梯度信息。
  • Momentum（动量）——引入"速度"的概念，让梯度下降像滚动的球一样，在梯度方向上一致时加速，在方向交替时减速。能有效穿过平坦区域和窄谷。
  • Adam（Adaptive Moment Estimation）——当前最流行的优化器。同时维护梯度的"一阶矩"（均值）和"二阶矩"（方差），为每个参数自适应地调整学习率。在大多数任务上表现优秀，是默认选择。
实践建议：如果你不确定用什么优化器，就用 Adam（学习率设为 0.001）。它是大多数任务的起点。如果追求极限性能，可以试试 SGD + Momentum + 学习率调度（学习率随训练逐渐降低）。AdamW 是 Adam 的改进版，将权重衰减独立出来，通常效果更好。

这一课的核心要点

  • 深度网络的每一层学习不同层次的抽象——从低级特征到高级语义，层次化地组织知识。
  • 反向传播 = 前向传播 + 链式法则 + 权重更新。它精确地计算了每个参数对损失的"贡献"。
  • 梯度下降沿着梯度的反方向更新权重。学习率决定了步长，太小太慢，太大发散。
  • Mini-Batch 梯度下降是实践中的标准：每次用一小批数据估计梯度，在效率和泛化之间取得平衡。
  • 过拟合是深度学习最大的敌人——防止过拟合需要数据增强、Dropout、早停、权重衰减等多种手段。
  • Adam 是目前最常用的优化器——它自适应地为每个参数调整学习率，在大多数任务上表现优异。
