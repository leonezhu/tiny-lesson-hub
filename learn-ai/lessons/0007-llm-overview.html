大语言模型 LLM

第 7 课 · ~12 minutes · 从 Transformer 到 GPT、Claude、Gemini

本课学完后，你将能够解释 LLM 的三个训练阶段（预训练、微调、对齐），理解"预测下一个词"如何在规模放大后涌现出推理和编程能力，并能说清缩放定律（Scaling Laws）的含义和它对 AI 发展的意义。

LLM 是什么？

大语言模型（Large Language Model, LLM）是以 Transformer 为架构基础，通过在海量文本上学习语言规律和世界知识的神经网络。GPT-4、Claude、Gemini、Llama 都属于 LLM。

LLM 的本质是一个概率分布建模器：给定前面的文字，预测下一个字/词是什么。但这个简单目标在规模足够大时，涌现出了令人惊讶的能力——推理、翻译、编程、对话。

关键洞察：LLM 学的不是"知识数据库"，而是一种世界模型——关于事物如何关联、因果关系如何运作的内部表示。当你问"地球到月球的距离"时，它不是查表，而是基于训练中见过的无数相关文本，概率性地生成正确答案。

LLM 的三个训练阶段

阶段一：预训练（Pre-training）—— 从海量文本中学习语言

做什么

在数千亿到十万亿 token 的文本上训练，目标是下一个 token 预测。模型看到文本的前面部分，猜下一个词是什么。猜错了就调整参数，猜对了就保留。

规模对比

  • GPT-3：5700 亿 token，1750 亿参数
  • PaLM：7800 亿 token，5400 亿参数
  • GPT-4：估计 10 万亿 token 以上，参数数量未公开（估计数万亿）

学到了什么

语法、事实知识、推理模式、不同领域的关联。预训练的损失函数遵循幂律衰减——每增加 10 倍数据，损失降低约 0.5-0.7 倍，说明数据越多越好，没有明显天花板。

阶段二：监督微调（SFT）—— 从"续写"到"回答问题"

问题

预训练好的模型只会"续写文本"。你给它一个开头，它接着写。但它不会主动回答问题、不会遵循指令。

解决

用人工编写的高质量指令-回答对进一步训练。比如：

  • 指令："请帮我写一封辞职信"
    回答："尊敬的领导，我因个人原因决定辞去目前的工作……"
  • 指令："Python 中如何读取 CSV 文件？"
    回答：给出 pandas.read_csv() 的代码示例

这一步让模型学会了"按照人类期望的方式回应"，而不仅仅是"顺着文字往下写"。

阶段三：人类反馈对齐（RLHF / DPO）—— 让输出符合人类偏好

问题

SFT 后的模型虽然能回答问题，但可能：回答太长、语气不好、提供有害信息。我们需要它更符合人类偏好。

两种方法

  • RLHF（人类反馈强化学习）：让人类对多个模型输出排序，训练一个"奖励模型"来模拟人类偏好，然后用强化学习（PPO 算法）优化模型。ChatGPT 用的就是这个方法。
  • DPO（直接偏好优化）：Claude 用的方法。绕过奖励模型，直接在偏好数据上优化——给模型看"好的回答"和"不好的回答"，让它学会区分。更简洁高效。

涌现能力：为什么"大"很重要？

LLM 最令人困惑的现象是涌现能力（Emergent Abilities）——某些能力在模型规模较小时完全不存在，但超过某个阈值后突然出现。

涌现的例子

  • 算术：小模型做加法一塌糊涂，大模型能做多位运算
  • 指令跟随：小模型不理解"请总结这篇文章"是什么意思，大模型能准确执行
  • 少样本学习：给几个例子就让模型学会新任务
  • 代码生成：从自然语言描述生成可运行的代码

这种现象与复杂系统中的相变类似——水在 0°C 突然结冰，铁在居里温度突然失去磁性。LLM 在参数量和训练量跨越某个临界点后，也发生了类似的"能力跃迁"。

重要提醒：涌现 ≠ 魔法。它仍然是统计学 + 工程学的结果。模型不是"突然变聪明了"，而是在足够大的规模下，之前分散的小能力整合成了更高级的能力。就像一滴水没有湿性，但亿万个水分子聚集起来就有了"湿润"的特性。

缩放定律（Scaling Laws）

缩放定律是 LLM 领域最重要的发现之一。它告诉我们：模型的性能可以通过简单地增加计算资源来预测和改善。

Kaplan 定律（2020）

训练损失与计算量呈幂律关系：L ∝ C-α（α ≈ 0.07）。这意味着计算量每增加 10 倍，损失降低约 0.7 倍。没有"智能奇点"，只有平滑的渐进改进。

Chinchilla 定律（2022）

在有限计算预算下，最优策略是让参数量和训练数据量以固定比例同步增长。一味堆参数而忽略数据量，或相反，都会导致性能次优。

这个发现直接改变了行业策略——之前大家拼命增大模型，后来发现数据量同样重要。

LLM 的能力边界

理解 LLM 能做什么、不能做什么，和了解它能做什么同样重要。

LLM 擅长的事

  • 文本生成、摘要、翻译、改写
  • 代码生成与调试
  • 问答（基于训练数据中的知识）
  • 创意写作、头脑风暴
  • 格式转换（JSON ↔ 表格 ↔ 自然语言）

LLM 不擅长的事

  • 事实准确性保证：LLM 会编造看似合理但完全错误的事实（"幻觉"）
  • 精确推理：复杂的多步逻辑推理仍然容易出错
  • 实时信息：不知道训练数据截止之后的事情
  • 确定性输出：同样的问题可能得到不同的答案
  • 真正的理解：LLM 没有意识或意图，它只是在统计意义上生成文本
核心理解：LLM 的"智能"是一种统计智能——它在海量数据中找到了模式和关联，但没有人类意义上的理解、意图或信念。把它当作一个极其强大的"模式匹配引擎"，而不是一个"思考的大脑"。
