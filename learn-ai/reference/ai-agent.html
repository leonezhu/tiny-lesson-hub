AI Agent — 速查卡

参考卡 · 源自 Lesson 9

Agent Loop 四步循环

感知 Observe→ 思考 Think→ 行动 Act→ 反思 Reflect↺
循环直到任务完成

核心组件

大脑（LLM）

推理核心，负责理解意图、制定计划、做出决策

工具箱（Tools）

搜索、API 调用、代码执行、文件读写等外部能力

记忆（Memory）

短期（对话历史）+ 长期（向量数据库 / RAG）

规划器（Planner）

任务分解策略，决定如何把大目标拆成子任务

两种典型模式

ReAct（推理+行动交替）

Thought → Action → Observation → Thought → Action → ...

适合：开放路径任务，需要实时决策

Plan-and-Execute（先规划后执行）

Plan: [1,2,3,4,5] → Execute 1 → Execute 2 → ...

适合：步骤明确的复杂任务

Agent vs 传统 AI

维度 传统 AI Agent
流程 一次请求一次响应 多轮循环
工具 通常没有 动态选择
错误处理 无法自修 反思修正

主流框架

  • LangChain / LangGraph — Python/JS，生产级首选
  • AutoGPT / BabyAGI — 早期自主 Agent 项目
  • OpenAI Function Calling — 轻量级 Agent 基础

常见挑战

  • 无限循环 → 设最大迭代次数
  • 成本失控 → 每轮 LLM 推理都消耗 token
  • 幻觉风险 → 编造不存在的工具/API
  • 调试困难 → 多轮路径难以追踪
