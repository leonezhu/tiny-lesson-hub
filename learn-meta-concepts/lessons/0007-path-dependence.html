路径依赖（Path Dependence）

Lesson 7 · ~10 minutes · Meta concept #7

通过本课，你将理解为什么历史上看似"不合理"的选择会长期锁定我们的制度、技术和文化——并且学会识别自己生活中的路径依赖，找到打破锁定的机会。

核心直觉

路径依赖的意思是：一个系统当前的状态，主要由它过去的经历决定，而不是由当前最优的选择决定。换句话说，"起点很重要"——甚至在某些情况下，起点决定了你永远到不了别的地方。

这个概念告诉我们：世界不总是往"最优"方向发展。有时候，只是因为一个偶然的小选择，整个系统就被锁定在了一条路径上，即使后来出现了更好的替代方案，也很难切换。

路径依赖的数学直觉

收益递增 + 转换成本 = 锁定效应

  • 使用的人越多 → 越有价值 → 更多人使用（正反馈）
  • 切换到新方案 = 重新学习 + 失去兼容性 + 承担风险
  • 即使新方案更好，锁定的惯性也可能让旧方案继续主导

经典案例：QWERTY 键盘

这是路径依赖最著名的例子。你可能不知道，QWERTY 键盘布局是一个"糟糕的设计"——它存在的初衷是为了降低打字速度，防止老式打字机的机械臂卡住。

然而，即使电动打字机出现后，QWERTY 已经被广泛使用：打字员习惯了它，培训机构教授它，键盘工厂生产它。到后来有人设计出更高效的键盘布局（如 Dvorak），转换成本已经高到无法克服——不是因为它更好，而是因为所有人都在用 QWERTY。

到 2024 年，QWERTY 依然占据全球超过 65% 的键盘市场份额。一个 19 世纪为机械打字机设计的布局，统治了 21 世纪的数字键盘。

关键教训：路径依赖的核心公式是 — 初始的微小优势 + 正反馈（收益递增） + 高转换成本 = 锁定。一旦锁定，即使有更好的选择，切换也极其困难。

路径依赖的四个要素

1. 设置（Settlement）— 初始状态

系统如何开始，往往不是最优的，甚至是随机的。

  • 美国为什么用英制单位（英尺、磅）？因为殖民地时期从英国继承了这套体系
  • 中国的繁体字→简化字改革，也是一个历史性的"设置点"

2. 正反馈（Positive Feedback）— 自我强化

一旦开始，系统会自我强化。

  • 越多人用 QWERTY → 越多人被教 QWERTY → 越多人生产 QWERTY 键盘
  • 越多人用 Windows → 越多软件为 Windows 开发 → 越多人买 Windows

3. 转换成本（Switching Cost）— 锁定

切换到新路径需要付出的代价。

  • 换手机操作系统 = 重新学习 + 丢失应用 + 联系人迁移麻烦
  • 中国公司从 Oracle 换到国产数据库 = 重新开发 + 测试 + 风险

4. 锁定（Lock-in）— 难以逆转

系统进入"锁定"状态，即使有更好的选择也难以切换。

  • 即使 Mac 和 Linux 更好，Windows 依然占据约 70% 桌面市场
  • 中国铁路使用 1435mm 标准轨距——跟美国一样，因为是 19 世纪从英国引进的

路径依赖不只是"坏"的

路径依赖有双刃剑效应。它既可以是锁定的陷阱，也可以是积累的引擎：

  • → 文化积累 中文几千年的文化传承本身就是一种路径依赖——越多人学，越有价值，但你也继承了古代的思维方式。
  • → 技术积累 Python 的生态系统（NumPy、Pandas、PyTorch）之所以强大，不是因为它是"最好的"语言，而是因为 20 年的积累形成了强大的正反馈。
  • → 制度积累 中国的科举制度延续了 1300 年，塑造了"知识改变命运"的文化路径依赖，直至今日的高考制度仍然体现这种逻辑。
现实洞察：当你觉得"某个东西明明不好但就是改不了"的时候，你可能遇到了路径依赖。解决路径依赖的关键不是"证明新方案更好"，而是降低转换成本或等待系统进入新的临界点。

如何在现实中应用

个人层面：识别你自己的路径依赖

  • 你的专业选择、职业方向，有多少是"因为已经学了/做了这么久"，而不是"因为现在做这个最优"？
  • 你用的手机/电脑/工具，是真的最好，还是只是因为习惯了？
  • 你的人际关系模式，有多少是童年经历的路径依赖？

决策层面：利用路径依赖创造优势

  • 在早期阶段抢占先机，建立正反馈循环（先入者优势）
  • 降低用户/学习者的转换成本，帮助用户从旧路径切换到你的新路径
  • 在锁定发生前，设计开放的系统（如开源软件）

练习：判断路径依赖

以下场景中，哪些体现了路径依赖？哪些不是？点击选择。

Q1
公司决定迁移到一个新的项目管理工具，因为团队发现旧工具效率低。
这个决定体现了路径依赖吗？
Q2
一家公司坚持使用 SQL Server，因为 DBA 团队只学过 SQL Server，即使 PostgreSQL 在性价比上更优。
这个情况体现了路径依赖吗？
Q3
一个初创公司在早期选择了 TypeScript，因为团队发现它减少了 bug，现在 TypeScript 的生态系统越来越完善，新项目都优先用 TS。
这体现了路径依赖吗？
Q4
一个人拒绝学习 AI 工具，因为他觉得"我用传统方法也能完成任务，没必要改变"。
这体现了路径依赖吗？
Q5
某个国家的法律体系基于英国普通法，因为该国曾经是英国殖民地。
这体现了路径依赖吗？
Q6
一个人换了新的操作系统后，发现新系统效率更高，用户体验更好。
这体现了路径依赖吗？
Q7
中国高铁采用了 350km/h 的运营标准，而日本新干线也是 320km/h 级别，因为它们都参考了欧洲的技术路线。
这体现了路径依赖吗？
Q8
一个城市选择了地铁作为主要公共交通系统，因为早期投资已经建成。即使轻轨成本更低，但因为地铁已经运营多年，市民已经习惯了，再改轻轨需要巨大的转换成本。
这体现了路径依赖吗？
有疑问？随时问我——我是你的元概念导师。不理解路径依赖和因果性的区别？为什么 QWERTY 布局至今未变？为什么有些路径依赖可以被打破？这些正是值得深入的问题。

延伸阅读：布莱恩·阿瑟《正反馈的魔法》（The Nature of Technology）——论述技术路径依赖和收益递增如何塑造经济演化；道格拉斯·诺斯《制度、制度变迁与经济绩效》——制度经济学中路径依赖的经典分析。
