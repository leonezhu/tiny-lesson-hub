路径依赖

Meta Concept #7 · 快速参考卡 · ~3 minutes

定义

路径依赖：一个系统当前的状态，主要由它过去的经历决定，而非当前最优的选择。

核心公式

初始微小优势 + 正反馈（收益递增） + 高转换成本 = 锁定

四个阶段

1. 设置（Settlement）

系统如何开始，往往不是最优的，甚至是随机的。

2. 正反馈（Positive Feedback）

越多人用 → 越有价值 → 更多人用。自我强化。

3. 转换成本（Switching Cost）

切换到新路径需要付出的代价：重新学习、失去兼容性、承担风险。

4. 锁定（Lock-in）

系统进入锁定状态，即使有更好的选择也难以切换。

经典案例

  • QWERTY 键盘 — 为降低打字速度设计，却成为全球标准
  • Windows 操作系统 — 即使 Mac/Linux 更优，仍占 ~70% 桌面市场
  • 1435mm 标准轨距 — 中国铁路跟美国一样，因为都从英国引进
  • Python 生态 — 不是因为"最好"，而是因为 20 年积累的正反馈
关键洞察：解决路径依赖不是"证明新方案更好"，而是降低转换成本或等待临界点。

应用方向

  • 个人 — 识别"因为已经做了太久"而非"因为最优"的决定
  • 创业 — 早期抢占先机建立正反馈，降低用户转换成本
  • 政策 - 在锁定发生前设计开放系统（如开源标准）
