大 O 记号（渐进上界）

f(n) = O(g(n)) ⇔ 存在常数 C, n0，使得对所有 n > n0，f(n) ≤ C · g(n)

复杂度等级速查

复杂度名称例子n=106
O(1)常数数组索引1
O(log n)对数二分查找20

P vs NP

P：能高效求解的问题（多项式时间算法可求解）

NP：能高效验证答案的问题（多项式时间算法可验证）

P ⊆ NP：总是成立（能求解自然能验证）

P = NP：未证明，多数人相信为假

NP-完全问题

NP 中最难的问题。任何一个存在多项式时间解法 ⇒ P = NP。

  • SAT（布尔可满足性）— 始祖，Cook-Levin 定理
  • TSP（旅行商问题）— 最优旅行路线
  • 背包问题 — 容量限制下的最优选择
  • 图着色 — 最少颜色方案
  • 子集和 — 是否存在子集和等于目标

三种应对策略

近似算法：保证解在最优解的 k 倍以内

启发式：通常效果好，不保证最优

专门化：利用问题特殊结构降复杂度

关键区分

可计算性 → 能不能算（图灵机）

复杂度 → 能不能高效算（大 O 记号）

一个算法可能"可计算但不可行"——如 O(2n)，n=100 时远超宇宙原子总数。
