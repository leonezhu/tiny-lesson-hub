信息论：信息、熵与压缩的本质

Lesson 6 · ~12 minutes · 信息度量基础

学完本课之后，你将能够理解\"信息\"的数学定义——为什么\"意料之外的消息\"比\"意料之中的消息\"包含更多信息，掌握熵的计算直觉，并能用信息论思维解释为什么压缩、通信、甚至加密都共享同一个核心原理。

信息到底是什么？

在日常生活中，\"信息\"意味着\"有意义的内容\"。但在信息论中，信息是一个精确的数学概念——它的核心直觉非常简单：信息的多少，取决于它有多意外。

举个例子：如果你的天气预报说\"明天太阳会从东边升起\"，这句话传递了多少信息？几乎为零——因为你早就知道了。但如果天气预报说\"明天下午三点会下冰雹\"，信息量就很大——因为你无法预测。

核心直觉：信息量与意外程度成正比

  • 高概率事件 = 低信息量。比如\"地球明天还存在\"——你知道这是真的，所以这句话几乎不包含信息。
  • 低概率事件 = 高信息量。比如\"地球明天不存在了\"——如果你收到这条消息，你会立即意识到它包含了巨大的信息量。
  • 确定性事件（概率=1）= 零信息量。你不会因为\"1+1=2\"得到任何新信息。

熵：不确定性的度量

克劳德·香农（Claude Shannon）在1948年创立了信息论，他给出的核心概念是熵（Entropy）。熵衡量的是一个随机变量的\"不确定程度\"或\"平均信息量\"。

直觉理解：熵越高，意味着你越无法预测下一个结果，因此每次观察到的结果带给你的平均信息量就越大。

香农熵公式
H(X) = −∑i P(xi) · log2(P(xi))

公式看起来很复杂，但直觉很简单：每个可能结果的\"信息量\"（−log P）乘上它出现的概率，然后把所有结果加起来。

两个世界：公平的硬币 vs 两面都是正面的硬币

让我们用具体的例子感受熵的力量。

情境一：一枚公平的硬币——正面和反面各50%概率。

公平硬币的熵
H = −[0.5 · log2(0.5) + 0.5 · log2(0.5)] = 1 bit

情境二：一枚两面都是正面的硬币——100%是正面。

假硬币的熵
H = −[1.0 · log2(1.0)] = 0 bits

结果很直观：假硬币的结果是确定的（你每次都能看到正面），所以每次\"观察\"它不需要任何信息来消除不确定性。而公平硬币每次都会给你1 bit的新信息——因为它完全不可预测。

关键洞察：熵 = 不确定性的度量 = 你为消除这种不确定性所需的信息量。0 bits = 完全确定，1 bit = 在两个等概率结果中做出选择。

信息量的单位：比特（bit）

\"比特\"不仅仅是\"0或1\"的缩写。它是一个信息量单位，表示\"在两个等概率选项中做出选择所需的信息量\"。

事件 概率 信息量（bits）
硬币正面 0.5 1.00
抛掷两次，HH 0.25 2.00
掷骰子是6 1/6 2.58
从100个数中选中 0.01 6.64
从10亿个数中选中 10−9 29.90

你会发现：从10亿个数中猜中一个，需要约30 bit的信息量——刚好对应一个30位的二进制数。这就是为什么二进制编码如此自然。

熵与压缩：为什么一切都能被压缩？

信息论最深刻的洞见之一：熵给出了无损压缩的极限。一个数据源的熵越低，它就越能被压缩。

霍夫曼编码：用最短的代码表示最频繁的事件

例子：假设你有一段文本，只包含字母 A、B、C、D，它们的出现频率分别是：

  • A: 50%
  • B: 25%
  • C: 15%
  • D: 10%

朴素编码（每个字母用2位表示）：

  • A=00, B=01, C=10, D=11 → 每个字母固定2位

霍夫曼编码（用短码表示高频字母）：

  • A=0 (1位) —— 最常见，用最短码
  • B=10 (2位) —— 次常见
  • C=110 (3位)
  • D=111 (3位)

效果：平均每个字母只需 0.5×1 + 0.25×2 + 0.15×3 + 0.10×3 = 1.6 bits，比2位少了20%。熵约为1.85 bits，所以1.6已经非常接近理论极限。

为什么ZIP能压缩文件？因为真实数据不是随机的——有重复模式、有频率分布。熵越低的文件（越有规律），越能被压缩。真正的随机数据（熵最大）是无法被无损压缩的。

为什么信息论重要？

香农的信息论不只是关于\"怎么压缩文件\"。它是整个数字时代的基础：

四大应用领域

  • 通信：5G、Wi-Fi、光纤——所有无线通信都在信道容量定理的框架下设计。香农证明了一个信道的\"容量上限\"，工程师的目标就是逼近它。
  • 加密：加密的本质是制造\"不确定性\"。一个好的加密算法应该让密文的熵尽可能高，让攻击者无法从密文中推断出任何信息。
  • 人工智能：信息熵是决策树算法（如ID3、C4.5）的核心——它用\"信息增益\"来决定哪个特征最能消除不确定性。交叉熵也是神经网络损失函数的重要组成部分。
  • 数据科学：特征选择、异常检测、聚类——所有这些方法都在做同一件事：在数据中寻找模式（低熵区域），忽略噪声（高熵区域）。

一个跨领域的直觉

想想热力学中的\"熵\"（混乱度）和信息论中的\"熵\"（不确定性）。它们本质上是同一个概念：

  • 物理熵：气体分子越混乱，系统熵越高 → 你越难预测某个分子的位置
  • 信息熵：事件越不确定，信息熵越高 → 你越难预测下一个结果

熵 = 你\"不知道\"的量。

常见错误

错误1：\"熵越高越好\"或\"熵越低越好\"。
熵本身没有好坏。在通信中，我们希望信号的低熵部分携带信息，高熵噪声被消除。在密码学中，我们希望密钥的熵足够高（不可预测），但信息的熵要低（能被压缩传输）。语境决定一切。
错误2：\"信息 = 有意义\"。
在信息论中，\"1+1=2\"和\"猫在草地上\"的信息量可能完全不同——这取决于你对它们的预期概率。信息论的\"信息\"是客观的数学量，不依赖于内容是否有意义。一条乱码如果完全随机，信息量极高（熵最大）；一条你早已知道的事实，信息量接近零。
错误3：\"熵就是\"混乱\"或\"无序\"\"。
虽然\"不确定性\"和\"混乱\"在直觉上相关，但熵是一个精确的统计量。一叠整齐的扑克牌和洗牌后的扑克牌，从信息论角度看，\"整齐的排列\"可能熵更低（因为你可以预测下一张牌），但这不是说\"整齐的扑克牌\"在物理上熵更低。两个领域的\"熵\"有数学联系，但不是一回事。
